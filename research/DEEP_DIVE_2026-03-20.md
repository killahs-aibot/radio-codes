# RadioUnlock Deep Dive Research Report
**Date:** 2026-03-20
**Researcher:** Tam (Subagent)
**Mission:** Find open-source/reverse-engineered algorithms for car radio unlock codes

## Executive Summary

The open-source community has **3 fully confirmed free algorithms** for car radio codes:
1. **Ford M Series** — 10×10 lookup table (OlegSmelov/ford-radio-codes)
2. **Renault/Dacia** — Precode modular arithmetic (yne/gist)
3. **Fiat VP1/VP2** — Nibble-extract lookup tables (mark0wnik/VP1-VP2-Toolkit)

**Everything else** requires either:
- Binary database lookup (Ford V Series ✅ radiocodes.bin exists)
- Forum-sourced pair databases (VW RCD, Opel/Vauxhall)
- Official portals (Honda ✅ free, Nissan ✅ free with VIN)
- EEPROM reading (the universal offline solution)
- Commercial services (PELock, dealer)

## Priority Brands — Detailed Findings

### 1. Honda — ❌ NO FREE ALGORITHM
**Status:** Proprietary, VIN-linked. **BUT: Official free portal exists.**

| Item | Detail |
|------|--------|
| Serial format | 8 digits (U12345678) or L + 4 digits |
| Code format | 5 digits |
| Free method | ✅ **https://radio-navicode.honda.com/** (VIN + serial) |
| Algorithm | ❌ Not publicly reversed |
| EEPROM | ✅ Works for all models |

**Key insight:** Honda's official portal is FREE and works for most vehicles. There's no need to reverse-engineer an algorithm — just guide users to the portal. The `honda.py` algorithm in the repo correctly points users to the portal.

**Acura:** Same system — https://radio-navicode.acura.com/

### 2. Peugeot/Citroen — ❌ MULTIPLE ALGORITHMS, NONE REVERSED
**Status:** No public algorithm. Multiple radio manufacturers use different schemes.

| Format | Radio Type | Status |
|--------|-----------|--------|
| BP prefix | Blaupunkt BP1/BP2 | ❌ Not reversed |
| 815 prefix | Valeo | ❌ Not reversed |
| 7 prefix | Siemens | ❌ Not reversed |

**Known vector:** `BP2 775(S) 9452352` → `2893` (Citroen Algo 1 from French Car Forum)

**Sources checked:**
- French Car Forum (frenchcarforum.co.uk) — has some vectors
- MHH AUTO RadioCodeDatabase — thousands of pairs
- CRUCC (carstereocode.com) — commercial tool, algorithm private

**Approach for RadioUnlock:**
1. Collect 100+ confirmed pairs from forums
2. Build lookup database (database approach)
3. For algorithm research: analyze CRUCC or collect enough pairs to spot patterns

**Peugeot/Citroen RT3/RT4/RNEG:** These use Blaupunkt BP1/BP2 algorithms. The CRUCC tool handles these but the algorithm is not public. Some forum posts suggest BP1/BP2 use lookup tables similar in complexity to Ford M Series.

### 3. Nissan — ❌ NO FREE ALGORITHM
**Status:** Proprietary encryption. **BUT: Official free portal exists.**

| Item | Detail |
|------|--------|
| Serial format | 12 hex chars (CY16C-1234567) |
| Code format | 4-5 digits |
| Free method | ✅ **https://radio-navicode.nissan.com/** (VIN + serial) |
| Algorithm | ❌ Not publicly reversed |
| EEPROM | ✅ Works for all models |

**Key insight:** Same as Honda — official portal is free and should be the primary recommendation. The `nissan.py` formula in the repo is a **placeholder guess** and should NOT be trusted.

**⚠️ WARNING:** The current `nissan.py` implementation uses an unverified formula:
```python
code = ((sum_of_hex_values * 3) + 7777) % 10000
```
This has NOT been confirmed with test vectors and likely produces WRONG codes.

### 4. Toyota ERC — ❌ NO FREE ALGORITHM
**Status:** Dealer-only ERC system. No public algorithm.

| Item | Detail |
|------|--------|
| Serial format | 16 alphanumeric chars |
| Code format | 4-6 digits (varies by model) |
| Free method | Dealer (may charge £10-20) |
| Algorithm | ❌ Not publicly reversed |
| EEPROM | ✅ Works for all models |

**PELock commercial:** Lists Toyota ERC as supported (paid).

**Key insight:** Toyota uses different algorithms per head unit manufacturer (Panasonic, Pioneer, Denso). No universal algorithm exists. The `toyota.py` in the repo correctly raises `NotImplementedError` and guides users to dealer/EEPROM options.

### 5. Alfa Romeo — ❌ RELATED TO FIAT VP1/VP2 BUT DIFFERENT
**Status:** Some Alfa models use Fiat VP1/VP2 algorithm, but many have proprietary algorithms.

| Item | Detail |
|------|--------|
| Known compatible | MiTo, Giulietta (VP1/VP2 based) |
| Unknown | 147, 156, 159, Brera, Spider (proprietary) |
| Algorithm | ❌ VP1/VP2 algo works for some models only |

**Key insight:** The `alfa.py` formula in the repo is a **placeholder guess** and should be removed or marked unverified. It uses:
```python
code = ((val * 9) + 0x1F3D) & 0xFFFF
```
This has NOT been verified with test vectors.

### 6. Chrysler/Dodge/Jeep Uconnect — ❌ VP2 PROPRIETARY
**Status:** Dodge Ram VP2 (Harman Kardon) is high-demand but algorithm not reversed.

| Item | Detail |
|------|--------|
| Known supported | PELock commercial — "Chrysler Dodge Ram VP2" |
| Algorithm | ❌ Related to Fiat VP1/VP2 but different |
| EEPROM | ✅ Works for all models |

**Key insight:** The `chrysler.py` formula in the repo is a **placeholder guess**:
```python
total = sum(ord(c) * (i + 1) for i, c in enumerate(serial))
code = ((total ^ 0x5A5A) + 3333) % 10000
```
This has NOT been verified with test vectors.

**Relationship to VP1/VP2:** The PELock listing groups VP2 with Fiat 250/500 VP1/VP2, suggesting related algorithms. However, VP2 may use a different implementation.

### 7. Blaupunkt (Multiple Brands) — ❌ ALGORITHM SUSPECTED BUT NOT REVERSED
**Status:** Blaupunkt radios use lookup-table algorithms (BP1, BP2) but none publicly reversed.

**Blaupunkt-covered brands:**
- Peugeot/Citroen (BP prefix serials)
- Opel/Vauxhall (GM0203... CAR2003)
- VW (some RCD models)
- Ford (some older models)
- Mercedes (Becker variants)

**The Challenge:** The MHH AUTO RadioCodeDatabase thread has thousands of Blaupunkt serial+code pairs, but the algorithm has not been extracted from them. BP1 and BP2 algorithms are suspected to use lookup tables similar to Ford M Series (10×10 or similar).

**Approach for RadioUnlock:**
1. Scrape MHH AUTO thread for 500+ BP serial+code pairs
2. Use the pairs to analyze and reverse the BP1/BP2 lookup table
3. This is the most promising path for new algorithm discovery
