# Ford V Series Radio Code Algorithm

## Overview
- **Brand:** Ford
- **Series:** V Series
- **Serial Format:** `V` + 6 digits (e.g., V123456)
- **Output Code:** 4 digits (0000-9999)
- **Works with:** Ford 3000, 4000, 4500, 5000, 6000 stereos
- **Models:** Escort, Fiesta, Focus, Mondeo, Transit

## Algorithm Type
Pre-computed lookup table (database-driven) rather than real-time calculation.

## Implementation

The V Series uses a different approach - a pre-generated database of all possible 
combinations. The serial number (without 'V' prefix, treated as a number) indexes 
directly into this table.

```c
// From fz_fordradiocodes Flipper Zero implementation
#define FILE_PATH "/ext/apps_data/fordradiocode/radiocodes.bin"

void process_data(FordRadioCodeApp* app) {
    char* endptr;
    long index = strtol(app->input_buffer + 1, &endptr, 10);
    
    size_t offset = (size_t)index * 2;
    if (app->input_buffer[0] == 'V') offset += 2000000;  // M serials offset
    
    // Read 2-byte result at offset from database file
    uint8_t buffer[2];
    stream_read(file_stream, buffer, sizeof(buffer));
    app->result = (int16_t)((buffer[1] << 8) | buffer[0]);
}
```

## Database Structure
- **M Series (0-999999):** Stored at offset `index * 2` in radiocodes.bin
- **V Series (0-999999):** Stored at offset `2000000 + (index * 2)` in radiocodes.bin
- **Format:** 2-byte little-endian unsigned integer per serial

## Algorithm (Theoretical)

Based on analysis of the database, V Series appears to use a similar but distinct 
lookup table algorithm from M Series. The exact algorithm has not been publicly 
reversed, but the pre-computed database covers all 1 million combinations.

```python
def ford_v_series_lookup(serial):
    """
    V Series uses a pre-computed database.
    This is the theoretical index calculation based on Flipper Zero code.
    """
    serial_num = int(serial[1:])  # Remove 'V', get numeric part
    
    # In the Flipper implementation, V codes are stored at offset 2000000
    # The actual algorithm to generate these values is not publicly known
    # The radiocodes.bin must be used for lookup
    
    # If you have the radiocodes.bin file:
    with open('radiocodes.bin', 'rb') as f:
        offset = 2000000 + (serial_num * 2)
        f.seek(offset)
        data = f.read(2)
        code = (data[1] << 8) | data[0]
        return f"{code:04d}"
```

the (undisclosed) algorithm. A radiocodes.bin with all 2 million entries exists 
in the Flipper Zero app repository.
